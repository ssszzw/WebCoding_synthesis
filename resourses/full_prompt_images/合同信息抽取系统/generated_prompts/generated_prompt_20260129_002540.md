# Role
你是一位精通 HTML5, CSS3, JavaScript (ES6+) 和 SVG 图形交互的高级前端开发工程师。

# Goal
编写一个单文件（`index.html`）的 Web 应用原型，完整复刻“合同信息抽取系统”的界面和交互逻辑。该系统不需要后端，所有数据和逻辑均在前端模拟。

# Visual Design & Layout Requirements
1.  **全局主题**：
    -   背景：使用紫色到蓝色的全屏渐变背景（参考：`linear-gradient(135deg, #6B73FF 0%, #000DFF 100%)` 或类似柔和的蓝紫色调）。
    -   字体：使用现代化无衬线字体（如 Roboto, Noto Sans SC, Microsoft YaHei）。
    -   整体风格：SaaS 风格，卡片式设计，圆角，轻微阴影。

2.  **界面状态一：文件上传页（初始状态）**
    -   居中显示一个白色卡片容器。
    -   标题：“合同信息抽取系统”，白色大号字体，位于卡片上方。
    -   卡片内容：
        -   一个带有虚线边框的拖拽区域（Dropzone）。
        -   中心图标：向上的上传箭头（使用 SVG）。
        -   主文本：“拖拽文件到这里或点击上传”。
        -   副文本：“支持 DOCX 和 PDF 格式文件”（灰色小字）。
        -   按钮：深蓝色背景的“选择文件”按钮。
    -   **交互**：点击“选择文件”或点击虚线区域，模拟加载 1 秒后，自动切换到“界面状态二”。

3.  **界面状态二：抽取工作台（主界面）**
    -   **布局**：左右分栏布局。左侧占 60%（文档预览），右侧占 40%（抽取信息面板）。整体包裹在一个大的白色圆角容器中，占据屏幕 90% 的区域。
    -   **左侧：文档预览区**
        -   顶部栏：左侧显示“合同文档”，右侧显示缩放控制（- / 100% / +）。
        -   内容区：模拟 Word 文档的样式（白底，页边距）。
        -   **文档内容（硬编码在 HTML 中）**：请完整包含图片中的示例文本：
            > 合同信息抽取测试文件
            > 合同编号：HT2024001
            > 合同签署日期：2024年1月15日
            > 合同签署地点：北京市朝阳区
            > 合同总金额：人民币50,000元整
            > 甲方信息：
            > 甲方名称：北京某某科技有限公司
            > 甲方地址：北京市海淀区中关村大街1号
            > ...（以及乙方信息和合同条款部分）
    -   **右侧：信息抽取面板**
        -   顶部栏：左侧“抽取信息”，右侧“重新抽取”按钮（点击返回上传页）。
        -   表单区域：包含多个字段组。每个字段包含：Label（左）、Input 输入框（中）、“定位”按钮（右，蓝色小方块）。
        -   **字段列表**：
            1.  合同签署日期
            2.  合同签署地点
            3.  合同总金额
            4.  甲方信息（作为小标题）：甲方名称、甲方地址
            5.  乙方信息（作为小标题）：乙方名称、乙方地址
    -   **视觉细节**：
        -   当某个输入框被选中（Focus）时，背景变淡黄色。
        -   文档中被抽取的对应文字高亮显示（蓝色背景）。
        -   **核心特效**：在文档的高亮文字和右侧对应的输入框之间，绘制一条虚线连接线（使用 SVG `pointer-events: none` 覆盖在最上层）。

# Functional & Interaction Requirements
1.  **数据模拟**：
    -   请在 JS 中定义一个 JSON 对象，存储每个字段对应的“值”以及该值在文档原文中的“文本内容”和“大致位置索引”。

2.  **定位功能（核心）**：
    -   当用户点击右侧表单项的“定位”按钮，或聚焦（Focus）到输入框时：
        1.  左侧文档自动滚动，确保对应文字在可视区域内。
        2.  左侧文档中的对应文字添加高亮样式（如深蓝色背景，白字）。
        3.  右侧输入框背景变黄，表示当前激活。
        4.  **动态连线**：动态计算左侧高亮文字 DOM 元素和右侧输入框 DOM 元素的屏幕坐标，在顶层 SVG 中绘制一条虚线连接两者。当滚动或调整大小时，连线应跟随更新。

3.  **原文修正功能**：
    -   当右侧某个输入框处于激活（Focus）状态时。
    -   用户在左侧文档区域用鼠标选中文一段新的文字（Selection API）。
    -   松开鼠标（mouseup）后，自动将选中的文本填充到当前激活的输入框中，并更新该字段的关联高亮位置。

# Technical Implementation Details
1.  **文件结构**：所有代码（HTML, CSS, JS）必须合并在一个 `index.html` 文件中。
2.  **SVG**：图标尽量使用内联 SVG 字符串，不要引用外部图片。
3.  **响应式**：保证在桌面端浏览器（宽度 > 1200px）下完美展示。
4.  **无依赖**：不使用 jQuery、Vue、React 等外部库，只使用原生 Vanilla JS。

-   请将 CSS 放在 `<style>` 标签中。
-   请将 JS 放在 `<body>` 底部的 `<script>` 标签中。
-   代码注释要清晰，解释清楚“坐标计算”和“连线绘制”的逻辑。
