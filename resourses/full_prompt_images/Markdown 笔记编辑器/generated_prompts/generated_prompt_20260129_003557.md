作为一个前端开发专家，请编写一个**单文件 HTML**（index.html）的 Markdown 笔记编辑器。该文件需要包含所有的 HTML 结构、CSS 样式（基于 Tailwind CSS）和 JavaScript 逻辑。

该编辑器需要精确复刻提供的参考图片中的视觉风格，并实现描述中的所有功能。具体要求如下：

### 1. 技术栈与库（通过 CDN 引入）
- **样式框架**：Tailwind CSS (v3.0+ Script 标签引入)。
- **图标库**：FontAwesome 或 Heroicons (SVG 形式嵌入或 CDN)。
- **Markdown 解析**：marked.js。
- **代码高亮**：Prism.js (包含默认主题和常用语言包)。
- **净化库**：DOMPurify (防止 XSS，可选但推荐)。

### 2. 布局与视觉设计
请实现一个现代、极简的**三栏式布局**（响应式，移动端隐藏侧边栏），配色方案需支持**浅色/深色主题切换**。

- **顶部导航栏 (Header)**：
  - 左侧：汉堡菜单按钮（控制侧边栏显隐）、应用标题 "Markdown 笔记编辑器"。
  - 右侧：深色/浅色模式切换按钮、导出按钮（蓝色）、导入按钮（绿色）。
  - 样式：固定高度，底部有边框，背景色需适配主题。

- **左侧侧边栏 (Sidebar) - 笔记列表**：
  - **搜索区**：顶部包含一个搜索框 "搜索笔记..."，圆角设计，带搜索图标。
  - **新建按钮**：明显的 "+" 悬浮按钮或顶部操作栏按钮。
  - **列表项**：
    - 显示笔记标题（若无标题则截取前20字）。
    - 显示相对时间（例如："刚刚"、"5分钟前"）。
    - 鼠标悬停显示 "编辑" 和 "删除" 图标。
    - 当前选中项需有明显的蓝色背景高亮（如图片所示）。
  - **滚动条**：自定义美化的细滚动条。

- **中间编辑区 (Editor Pane)**：
  - **工具栏**：固定在编辑区顶部。包含功能图标：加粗 (B)、斜体 (I)、H1/H2/H3、链接、图片、代码块、引用、列表。
  - **输入框**：占满剩余空间，无边框 textarea，等宽字体，支持大量文本输入。
  - **交互**：点击工具栏按钮时，应在光标位置插入对应 Markdown 语法，并保持光标焦点。

- **右侧预览区 (Preview Pane)**：
  - **样式**：必须模拟 **GitHub Markdown CSS** 风格（白色背景，特定的标题边框，引用块样式，表格样式等）。
  - **功能**：实时渲染左侧输入的 Markdown 内容。

- **拖拽分割线 (Resizer)**：
  - 位于编辑区和预览区之间。
  - 鼠标拖拽可实时调整左右两栏的宽度比例。
  - 样式为一条细灰线，悬停时鼠标变为 `col-resize`。

### 3. 核心功能逻辑 (JavaScript)

请在一个 `<script>` 标签内实现以下完整的业务逻辑：

**A. 数据管理 (Data Persistence)**
- 使用 `localStorage` 存储笔记数组。
- 数据结构应包含：`id` (UUID), `title`, `content`, `createdAt`, `updatedAt`。
- 实现自动保存机制（防抖 debounce 处理，输入停止后 500ms 自动保存）。
- 初始化时检查 localStorage，若无数据则加载一篇包含丰富格式（标题、列表、代码、表格）的“欢迎说明”默认笔记。

**B. 笔记操作 (CRUD)**
- **新建**：清空编辑器，创建新 ID，并自动聚焦。
- **删除**：点击删除图标弹出确认提示（confirm），确认后移除并刷新列表。
- **搜索**：根据搜索框内容实时过滤左侧笔记列表（匹配标题或内容）。
- **切换**：点击列表项，编辑器和预览区加载对应内容。

**C. Markdown 渲染与增强**
- 配置 marked.js 支持 GFM (GitHub Flavored Markdown)。
- 每次渲染后，强制触发 Prism.js 的 `highlightAll()` 以实现代码高亮。
- 能够正确渲染表格、引用、任务列表 `[ ]`。

**D. 文件导入导出**
- **导出**：点击导出按钮，将当前笔记内容下载为 `.md` 文件。
- **导入**：点击导入按钮，触发 `<input type="file">`，读取 `.md` 文件内容并创建新笔记。

**E. 交互细节**
- **快捷键**：支持 `Ctrl+S` (手动保存), `Ctrl+B` (加粗), `Ctrl+I` (斜体)。
- **字数统计**：在编辑器底部或角标显示当前字数。
- **响应式**：在移动端（<768px）隐藏预览区，侧边栏变为抽屉式或全屏切换。

### 4. 代码结构要求
- 所有代码必须合并在**唯一的 index.html** 文件中。
- CSS 需主要使用 Tailwind utility classes，对于 PrismJS 主题和 Markdown 自定义样式（如类似 GitHub 的排版），请在 `<style>` 标签中编写必要的自定义 CSS。
- 代码需整洁、有注释，确保复制后直接在浏览器打开即可完美运行。
