# AI虚拟试衣功能前端开发提示词

请创建一个包含完整 HTML、CSS 和 JavaScript 的单文件 (`index.html`)，实现基于 AI 的虚拟服装试穿界面。

### 1. 整体界面风格与布局
*   **主题**：深色模式（Dark Mode），背景色使用深灰/黑色（如 `#111`），文字为浅灰/白色，强调现代科技感。
*   **布局**：左右分栏结构。
    *   **左侧边栏（宽度约 320px-360px）**：操作控制区。
    *   **右侧主区域**：结果展示区，居中显示生成的试衣效果图。

### 2. 左侧控制区功能模块
*   **顶部切换 Tab**：包含“通用生成”和“服装试穿”（默认选中高亮，样式如截图中的白色胶囊状）。
*   **试穿描述输入框**：Textarea，Placeholder 提示“描述您希望的试穿效果...”，限制字符数。
*   **图片上传区域（核心）**：
    *   **人物照片**：上传框 + 预览缩略图，标签“人物照片”。
    *   **服装照片**：上传框 + 预览缩略图，标签“服装照片”。
    *   *交互*：点击上传或拖拽，上传后显示图片预览。
*   **提示词建议**：一行图标按钮（画板、风格等），仅作视觉展示。
*   **底部操作栏**：
    *   设置折叠面板。
    *   **生成按钮**：宽大的白色圆角按钮，文字“生成试穿效果”，右侧显示“20 积分”。点击触发 API 请求，显示 Loading 状态。

### 3. 右侧展示区视觉细节
*   **标题**：顶部显示“演示图集”。
*   **提示信息**：灰色圆角矩形框，内含绿色“你的生成结果”标签及说明文字。
*   **主预览窗口**：
    *   大尺寸图片容器，默认显示占位图或引导图。
    *   生成成功后，平滑淡入显示结果图片。
    *   右下角添加半透明水印文字“AI生成”。
*   **背景装饰**：背景可添加微弱的径向渐变或科技感线条纹理。

### 4. 逻辑与 API 实现要求
*   **API 集成**：
    *   使用 `fetch` 模拟调用火山引擎 API。
    *   **Endpoint**: `https://ark.cn-beijing.volces.com/api/v3/images/generations`
    *   **Method**: `POST`
    *   **Model**: `doubao-seedream-4-0-250828`
    *   **Hidden Prompt**: 代码中内置一个高质量 Prompt（如 "A realistic photo of a model wearing the uploaded clothing, high fashion, 4k texture"），**不要**显示在前端界面上，并在请求时自动拼接到 Payload 中。
    *   **Payload 结构**：参考描述中的 `curl` 示例，将用户上传的两张图片（转为 URL 或 Base64 模拟）放入 `image` 数组。
*   **交互逻辑**：
    *   验证两张图片是否已上传。
    *   点击生成 -> 按钮变更为“生成中...” -> 等待模拟延时 -> 显示结果图。
    *   若无真实后端，请在 JS 中写死一个模拟的返回结果（使用占位图 URL）以展示流程。

### 5. 代码输出规范
*   将所有 CSS 样式（使用 Flexbox/Grid 布局，CSS Variables 定义颜色）、HTML 结构、JavaScript 逻辑（事件监听、API 请求模拟）合并在唯一的 `index.html` 文件中。
*   确保代码无外部依赖（字体可用系统默认），直接在浏览器打开即可运行 UI 交互。
