请为一个Web前端开发专家编写一份详细的Prompt，目标是根据以下要求和视觉参考生成一个完整的单文件 HTML 项目。

---

**角色设定：**
你是一位精通 HTML5, CSS3 和原生 JavaScript 的高级前端开发工程师和 UI 设计师。

**任务目标：**
创建一个名为“魔法调酒师 (Magic Bartender)”的交互式网页应用。该应用需复刻提供的设计图效果，允许用户通过选择心情、调整口味参数和选择原材料，来生成一杯虚拟的“专属饮品”。

**文件要求：**
1.  **单文件输出：** 所有的 HTML 结构、CSS 样式和 JavaScript 逻辑必须合并在同一个 `index.html` 文件中。不要引用任何外部 CSS 或 JS 库（如 Bootstrap, jQuery 等），也不要使用外部图片链接（图标请使用 Emoji 或内联 SVG）。
2.  **响应式设计：** 页面应在桌面端完美展示（类似设计图的双栏布局），在移动端自动调整为单栏布局。

**详细需求说明：**

### 1. 视觉设计与布局 (CSS)
*   **整体背景：** 使用深紫色到浅紫色的线性渐变背景（参考色值：`linear-gradient(135deg, #667eea 0%, #764ba2 100%)`）。
*   **头部 (Header)：**
    *   居中显示标题“🍸 魔法调酒师 🍹”，字体较大，白色，带有微弱的发光效果。
    *   副标题：“创造属于你的专属饮品，调出心情的味道”，白色，字号稍小。
*   **主体容器：**
    *   页面主体分为左右两个面板，使用 Flexbox 或 Grid 布局。
    *   每个面板都是一个白色卡片，具有较大的圆角（约 20px）和柔和的阴影（box-shadow），内边距适中。
*   **字体：** 使用现代无衬线字体（如 sans-serif, PingFang SC, Microsoft YaHei）。

### 2. 左侧面板：定制区域 (功能交互)
左侧面板标题为“🎨 定制你的饮品”。包含以下模块：

*   **模块 A：心情选择 ("今天的心情如何？")**
    *   展示 6 个心情选项：开心、放松、浪漫、充满活力、忧郁、冒险。
    *   **UI表现：** 每个选项由 Emoji 图标和文字标签组成。
    *   **交互：** 默认灰色或透明背景。选中项应高亮显示（例如：深紫色背景、白色文字、稍大的图标缩放效果）。支持单选。
*   **模块 B：口味偏好 ("调整口味偏好")**
    *   包含 5 个滑动条（Input Range）：甜度、酸度、苦度、辣度、清爽度。
    *   **UI表现：** 每个滑动条左侧有对应的 Emoji（如 🍓, 🍋, ☕, 🌶️, ❄️），右侧实时显示数值（1-10）。
    *   **样式：** 自定义滑动条样式，滑块（thumb）设为紫色圆形，轨道（track）为浅灰色。
*   **模块 C：材料选择 ("选择主要材料")**
    *   分为 4 个子类目：基酒、果汁、利口酒、装饰。
    *   每个类目下有若干选项（例如：基酒含伏特加、金酒、朗姆酒等）。
    *   **UI表现：** 使用标签（Tag/Chip）样式。未选中为浅灰背景，选中为蓝紫色背景+白字。
    *   **交互：** 点击切换选中状态。每个类别至少支持单选，或多选（逻辑需自洽）。
*   **底部按钮：**
    *   **“重置选择”：** 幽灵按钮（白色背景，蓝色边框和文字），点击重置所有输入。
    *   **“✨ 开始调酒”：** 显眼的主操作按钮（深紫色渐变背景，白色文字），点击触发生成动画。

### 3. 右侧面板：结果展示 (视觉特效)
右侧面板标题为“🍹 你的专属饮品”。初始状态可以显示占位符或空杯子。点击“开始调酒”后更新内容：

*   **饮品可视化 (核心特效)：**
    *   使用 CSS 绘制一个杯子形状（U型容器）。
    *   **液体动画：** 当点击“开始调酒”时，液体应从杯底向上产生“注满”的 CSS 动画效果。
    *   **动态颜色：** 液体的颜色应根据左侧选择的“材料”或“心情”动态改变（例如：选朗姆酒+橙汁=橙黄色；选忧郁+蓝橙酒=深蓝色）。
    *   **装饰物：** 根据选择的装饰（如薄荷叶、柠檬片），在杯口上方通过绝对定位显示对应的 Emoji 或图形。
*   **饮品信息：**
    *   **名称：** 随机或根据算法生成一个富有诗意的名字（如：阳光微笑、午夜飞行、蓝莓之夜）。
    *   **描述：** 一段简短的生成文案（如：“这款饮品充满了阳光的味道...”）。
    *   **配方表：** 列出所选材料及随机生成的毫升数（例如：朗姆酒 36ml）。
*   **Toast 通知：**
    *   当制作完成时，在页面底部中央弹出一个黑色的 Toast 提示框：“✨ 你的专属饮品已调好！”，3秒后自动消失。

### 4. JavaScript 逻辑实现
*   **数据结构：** 定义好心情、材料、口味的数据对象。
*   **生成逻辑：**
    *   监听“开始调酒”按钮点击事件。
    *   验证是否至少选择了一种基酒和一种果汁（如果未选，提示用户）。
    *   根据选中的心情和材料，计算最终饮品的颜色（RGB值混合或预设映射）。
    *   随机生成饮品名称和描述文案。
    *   触发右侧面板的 DOM 更新和 CSS 动画。
*   **重置逻辑：** 清空所有状态并恢复默认视图。

### 5. 代码质量要求
*   HTML 语义化。
*   CSS 类名命名规范（BEM 或清晰的语义命名）。
*   代码中需包含适当的中文注释，解释关键逻辑。
*   确保布局美观，间距合理，还原度高。
