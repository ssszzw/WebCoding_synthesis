# 开发指令：构建反应力测试 H5 小游戏

请根据以下详细规格说明，编写一个包含 HTML、CSS 和 JavaScript 的单文件 `index.html`，实现一个“反应力测试小游戏”。

## 1. 整体视觉风格与布局
*   **背景风格**：使用紫色到蓝色的线性渐变背景（参考色值：类似 `#6a11cb` 到 `#2575fc`），营造现代、沉浸式的氛围。
*   **排版布局**：
    *   内容垂直居中排列。
    *   顶部为游戏标题，白色无衬线字体，带有轻微阴影。
    *   **数据状态栏**：标题下方并排两个胶囊状（Pill-shaped）容器，背景为半透明白色（玻璃拟态效果），分别显示“当前轮次：X / 10”和“平均反应时间：XXXX ms”。
    *   **控制按钮**：位于状态栏下方，左对齐或居中。颜色为珊瑚红/橙红色（如 `#ff6b6b`），圆角按钮，鼠标悬停时有高亮或轻微缩放效果。文案为“开始游戏”。
    *   **游戏主区域**：位于底部的大块区域。
        *   外观：白色背景，大圆角（约 12px-20px），带有柔和的投影（Box-shadow）以体现层级感。
        *   功能：这是测试反应的交互区域，黑色方块将在此区域内随机生成。

## 2. 交互与功能逻辑
*   **游戏流程**：
    1.  **初始状态**：显示标题、状态栏（初始清零）、“开始游戏”按钮。游戏区域空白。
    2.  **开始游戏**：点击按钮后，按钮变为禁用或隐藏状态。
    3.  **随机等待**：进入每一轮前，系统随机等待 1~3 秒。
    4.  **目标出现**：
        *   在白色游戏区域内**随机位置**生成一个深灰色/黑色圆角方块（约 50x50px）。
        *   确保方块完全位于白色区域内部，不会溢出边界。
    5.  **玩家点击**：
        *   玩家点击方块瞬间，记录从“方块出现”到“点击发生”的时间差（毫秒）。
        *   方块立即消失。
    6.  **循环逻辑**：
        *   更新顶部的“当前轮次”和实时“平均反应时间”。
        *   共进行 **10轮** 测试。
    7.  **结算状态**：
        *   10轮结束后，弹出一个模态框（Modal）或覆盖层显示最终成绩。
        *   **成绩内容**：最终平均时间、最快一次时间、最慢一次时间。
        *   **图表展示**：使用 Canvas 或简单的 CSS 柱状图，展示这10次反应时间的波动趋势（反应偏差图表）。
        *   提供“重新开始”按钮。

## 3. 技术实现要点
*   **文件结构**：所有代码（HTML 结构、CSS 样式、JS 逻辑）必须合并在一个 `index.html` 文件中。
*   **CSS**：
    *   使用 Flexbox 或 Grid 进行居中布局。
    *   必须适配移动端（Responsive Design），在手机屏幕上游戏区域宽度自适应，方块大小适中。
    *   使用 CSS 变量管理颜色，方便调整。
    *   按钮和方块添加 `cursor: pointer` 和简单的过渡动画（Transition）。
*   **JavaScript**：
    *   使用原生 JavaScript，无需构建工具。
    *   如果需要绘制图表，请通过 CDN 引入轻量级图表库（如 Chart.js），或者手写简单的 SVG/Canvas 绘图逻辑。
    *   确保随机坐标计算逻辑严谨：`Math.random() * (containerWidth - boxSize)`。
    *   添加防止误触逻辑（例如：如果在方块出现前点击白色区域，可以给予提示或判定无效，但这步为可选优化，以基础功能为主）。

## 4. 动画与特效细节
*   **方块出现**：瞬间出现，或者极短的淡入效果（Fade-in < 100ms），以免影响反应测试的准确性。
*   **点击反馈**：点击方块时，方块可以有一个微小的缩放动画（`transform: scale(0.9)`）表示被击中。
*   **结算弹窗**：带有淡入和上浮的进入动画。
