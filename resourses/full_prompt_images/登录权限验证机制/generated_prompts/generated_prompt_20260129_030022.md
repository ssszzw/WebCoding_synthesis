# 前端开发提示词：合同信息抽取与权限管理系统

请你作为一名资深的前端全栈工程师，编写一个包含 HTML, CSS 和 JavaScript 的单文件 (`index.html`)，实现一个**合同信息抽取系统**。该系统需包含完整的用户注册登录流程、文件上传界面以及核心的合同信息抽取与校对工作台。

### 1. 整体视觉风格与布局
*   **色调**：主色调使用蓝紫色渐变背景（参考 `linear-gradient(135deg, #667eea 0%, #764ba2 100%)`），营造现代、科技感的氛围。内容区域使用白色或半透明磨砂玻璃效果。
*   **字体**：使用系统默认的无衬线字体（如 Inter, Roboto, PingFang SC），确保清晰易读。
*   **响应式**：界面需在桌面端完美展示，元素居中对齐，布局协调。

### 2. 功能模块与交互细节

#### A. 权限验证模块（登录/注册）
此模块需复刻提供的图2布局。
*   **布局**：左右分栏设计。
    *   **左侧（40%）**：展示系统品牌信息。
        *   标题：“合同信息抽取系统”。
        *   副标题：“智能识别合同关键信息”。
        *   功能列表（带图标）：支持DOCX和PDF格式、智能信息抽取、原文定位高亮、支持手动校正、安全用户认证。
    *   **右侧（60%）**：白色卡片区域，包含表单。
        *   **登录状态**：标题“欢迎回来”，显示“还没有账户？立即注册”链接。表单包含：邮箱地址、密码、记住我（复选框）、忘记密码（链接）、登录按钮。底部显示隐私协议提示。
        *   **注册状态**：点击“立即注册”切换视图。表单包含：邮箱地址、设置密码、确认密码、注册按钮、返回登录链接。
*   **逻辑**：
    *   使用 `localStorage` 模拟后端数据库，存储用户的邮箱和密码。
    *   注册时检查邮箱是否已存在，密码是否一致。
    *   登录时校验邮箱和密码，成功后跳转至主系统，并保存登录态。

#### B. 主系统 - 文件上传页
此模块需复刻提供的图1布局。
*   **顶部栏**：左侧显示系统名称“合同信息抽取系统”，右侧显示“欢迎，[用户邮箱]”及“退出”按钮（透明背景，白色文字）。
*   **中心区域**：一个大的白色圆角卡片，带有虚线边框的拖拽区域。
    *   图标：巨大的上传图标（向上箭头）。
    *   主文案：“拖拽文件到这里或点击上传”。
    *   副文案：“支持 DOCX 和 PDF 格式文件”。
    *   按钮：紫蓝色实心按钮“选择文件”。
*   **交互**：
    *   支持拖拽文件进入虚线框（增加 `dragover` 高亮效果）。
    *   点击按钮触发文件选择器。
    *   文件选中后，显示模拟的“正在分析...”加载动画（Loading Spinner），2秒后自动进入“工作台页面”。

#### C. 主系统 - 合同抽取工作台（核心功能）
由于是纯前端演示，无需真实解析二进制文件，请使用**模拟数据**实现以下效果：
*   **布局**：左右两栏布局。
    *   **左侧（原文展示区）**：
        *   模拟显示一份HTML格式的合同范本（包含甲方、乙方、金额、日期等文字）。
        *   样式需像一份真实的A4文档（白色背景，阴影，衬线字体）。
    *   **右侧（抽取结果校对区）**：
        *   固定宽度的侧边栏，标题“抽取结果”。
        *   **表单字段**：甲方、乙方、甲方地址、乙方地址、合同编号、合同签署日期、合同签署地点、合同总金额。
*   **核心交互（高亮定位）**：
    *   **点击定位**：当用户点击右侧表单中的某个输入框（如“甲方”）时，左侧文档中对应的原始文字（如“深圳市XX科技有限公司”）应自动高亮显示（背景色变黄或变紫），并自动滚动文档使该位置可见。
    *   **手动校正**：用户可以直接修改右侧表单中的内容。
    *   **保存**：底部提供“保存结果”按钮，点击后弹出“保存成功”提示。

### 3. 技术实现要点
*   **单文件结构**：CSS 样式写在 `<style>` 标签中，JavaScript 逻辑写在 `<script>` 标签中，确保直接打开 `index.html` 即可运行。
*   **图标**：使用 SVG 字符串直接嵌入代码中，不依赖外部字体图标库（如 FontAwesome）。
*   **状态管理**：使用简单的 JavaScript 变量控制当前显示的视图（`view: 'login' | 'register' | 'upload' | 'workspace'`）。
*   **模拟数据**：在 JS 中预定义一份包含 HTML 标签的合同文本，并在标签上添加 `id` 或 `data-field` 属性，以便实现点击输入框时的定位功能。

### 4. 输出要求
请直接生成完整的 `index.html` 代码，无需解释，代码结构需包含：
1.  `<!DOCTYPE html>`
2.  `<html>` 包含 `<head>`（CSS）和 `<body>`（HTML结构 + JS逻辑）。
3.  确保 CSS 样式美观，复刻截图中的渐变、阴影和圆角细节。
4.  确保 JS 逻辑包含完整的注册、登录、模拟上传、模拟抽取和高亮定位功能。
